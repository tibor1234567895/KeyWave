# Contributing to KeyWave

Thanks for your interest in KeyWave.

## Ground Rules
- Be respectful and constructive.
- Keep changes focused and well-described.
- Follow the project coding style and PR templates.

## Pull Requests
- Describe the problem and the solution.
- Include tests for new logic where possible.
- Update docs when behavior changes.

## Development
- Android Studio recommended
- Kotlin + Compose
